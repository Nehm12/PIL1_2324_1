Structure du Projet

Langages de programmation et Framework

Cette application a été développé avec 4 langages de programmation, 2 frameworks et quelques autres technologies web :

  • HTML
  • CSS
  • Javascript
  • Python
  • Django
  • BootStrap
  • AJAX
  • Datatables

Structure en Django

Cette application dispose de trois sous applications Django pour gérer chaque fonctionalités :

  • Account pour les pages rapides d'acceuil de connexion des utlisateurs
  • Chat pour a messagerie Instantanée (qui reste des imperfections à corriger)
  • Suggestion des utlisateurs
  • Notifications (Que nus avons pas pu achevé vu le temps qui nous ai donné)

Database

Cette application utilise les modèles Django, qui font partie du framework Django, pour interagir avec la base de données. Nous avons choisi MySQL comme système de gestion de base de données pour stocker et gérer les données.

Instructions de déploiement

Il est nécessaires de suivres les instruction suivantes pour assurer le bon fonctionnement de l'application :

Les Premiers Pas

  1. Obtenir le projet sur GitHub
    • Cloner ou Telechargez le zip du projet depuis Github et extraire : Cliquez pour le lien du projet GitHub sur la branche second
  2. Une fois sur votre PC, vous aurez le projet dans un dossier PIL1_2324_1
    • Ouvrez un terminal dans le dossier PIL1_2324_1 et lancez les commandes suivantes:

      • NB: Si vous n'avez pas pipenv, installez le en utilisant 

      pip install pipenv

    • Dans le terminal, effectuez les commandes suivantes
      • pipenv lock

            • Si vous rencontrez des erreurs, faites ces trois commandes:

          • pipenv clean
          • pipenv install --skip-lock
          • pipenv lock
      • pipenv sync
      • pipenv shell
    • Configuration du Database
      • Creez une base de données sur votre PC (avec PHPMyAdmin, MySQL Workbench ou d'autres) et nommez le rqt
      • Importer la structure en utilisant le fichier rqt.sql donné
        • Si vous utilisez mysql cli (terminal)
          mysql -u username -p rqt < chemin/vers/rqt.sql
          Remplacez username par votre nom d'utilisateur et chemin/vers/rqt.sql par le chemin menant à ce fichier

        • Si vous utilisez PHPMyAdmin, ce dernier vous propose une interface graphique pour importer le fichier sql
      • Dans le dossier PIL1_2324_1, lancez la commande: 
        python Rencontre/Rencontre/config.py

        Renseignez votre nom d'utilisateur, mot de passe, hote, et port.

        NB: Entrez O ou N pour répondre à Voulez vous changer ...

      • Django utilise un SECRET_KEY dans ces applications et c'est mieux si ces clés sont uniques. Alors, lancez la commande suivante pour générer votre SECRET KEY 
        python Recontre/Rencontre/generate_key.py
    • Lancement du projet

      • Lancez les deux commandes suivantes pour lancer l'application

      Naviguez dans le dossier Account

      • pipenv shell
      • cd Account

        • Lancez le serveur

      • python manage.py runserver port

        • Remplacez port par le port que vous voulez ou laissez vide

    • Accedez au projet dans un navigateur depuis 
      localhost:port
      ou 
      localhost
      (si vous n'avez par spécifié de port)


      Vous pouvez egalement utiliser 
      127.0.0.1

      • Apres le Lancement du projet, vous verrez aussi l'url de l'application

5. Conclusion

En conclusion, le projet de crétaion d'Application Web de rencontre. Nous avons réussi à développer un système fonctionnel et performant, répondant aux besoins spécifiques du projet Integrateur. Ce projet a permis d'améliorer la rencontre et l'extensions des connaissance de ces utlisateurs .

6. Recommandations

Afin d'améliorer encore davantage le site de gestion des emplois du temps, nous recommandons les actions suivantes :

  • Continuer à recueillir les retours des utilisateurs et à les intégrer dans les mises à jour futures.
  • Renforcer la sécurité du système en mettant en place des mécanismes d'authentification et de contrôle d'accès plus avancés.
  • Améliorer les performances en optimisant les requêtes et en utilisant des techniques de mise en cache.
  • Élargir la portée du projet en développant des applications mobiles pour un accès facile depuis les smartphones et les tablettes.
